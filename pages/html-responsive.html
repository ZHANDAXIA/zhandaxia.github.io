响应式布局之viewport

响应式布局的两个关键点

在做响应式布局的时候有两个关键因素是缺一不可的,其一就是@media,另外一个就是viewport

viewport翻译成中文就是:视窗,视口,也就是看到的窗口.在pc端它的默认宽度就是窗口宽度,对布局没什么影响,但是到了移动端问题就大了.在移动端viewport绝大部分默认值都为980(除了个别ipad pro等分辨率特别高的设备).所以viewport的默认值和屏幕实际尺寸不相等,这就会导致移动端页面出现各种问题.解决这个问题的办法就是人为的设置viewport属性.

<meta name="viewport" content="width=device-width,initial-scale=1,maximum-scale=1,minimum-scale=1,user-scalable=0">

在<head>内设置此meta标签可以很好的解决移动端的问题.

viewport的属性

viewport共有6个属性:

  • width
  • height
  • initial-scale
  • maximum-scale
  • minimum-scale
  • user-scalable

width

width决定viweport的宽度,可以设置成固定大小,但是为了兼容各种大小不一的移动端设备一般都设置成width=device-width.让viewport的宽度等于设备的宽度防止溢出或过小.

height

height决定viewport的高度,一般不用设置.

initial-scale

initial-scale决定viewport的初始缩放比例,一般设置为initial-scale=1,解决设备从竖屏转换成横屏是所出现的缩放问题.

maximum-scale

maximum-scale决定可以缩放的最大比例,可以设置一个大于等于1的数值,如果设置为maximum-scale=1就表示不能缩放.

minimum-scale

minimum-scale决定可以缩放的最小比例,可以设置一个小于等于1的数值,如果设置为minimum-scale=1就表示不能缩放.

user-scalable

user-scalable决定是否允许用户缩放,设置为user-scalable=1或user-scalable=yes表示允许缩放,设置为user-scalable=0或user-scalable=no则表示不允许缩放.
